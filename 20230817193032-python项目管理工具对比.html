python项目管理工具对比

前言

Python 没有官方的项目管理工具，它只有包管理工具 pip 用于各种包的安装删除等操作。但在开发过程中，不同的项目都依赖不同的包，如果把这些项目依赖的包都安装在一个环境下，那么总是会遇到包之间冲突的问题。比如不同的项目需要不同版本的包、有些包不能同时安装、 Python 版本的依赖不同、与操作系统软件管理工具的冲突等。

所以进行 Python 开发时一个优秀的项目管理工具是必要的，接下来我就来简单讲讲我使用和听说过的项目管理工具。

pip

其实只使用 pip 肯定可以解决一个项目中软件包的安装问题，它只是不能处理多个项目之间的问题。

甚至项目管理工具不能处理的包，最后只能使用 pip 进行管理。

虚拟环境

介绍

针对 pip 不能管理多个项目之间包依赖的问题，虚拟环境这种技术就应运而生了。像 virtualenv, virtualenvwrapper, pyenv, venv 都可以创建虚拟环境。但我只推荐 venv ，因为它是官方出的库，不需要安装。

原理

venv 就是把系统 Python 链接复制一份到 venv 的环境，用命令 source activate 进入一个 venv 环境时， venv 会修改相关环境变量，让命令 python 和 pip 均指向当前的 venv 环境。

这样每个虚拟环境都可以任意的调整包与 Python 版本，不同的环境之间不会进行影响，所以对每个项目创建自己的虚拟环境就可以解决项目之间包的依赖冲突问题了。

方法

其实使用 venv 加 pip 可以解决所有的问题，只不过相对项目管理工具来说比较繁琐。但有相当多的项目使用了这种方式，它们把所有需要的包放入 requirements.txt ，虚拟环境交由用户自己决定，然后使用 pip install -r requirements.txt 就可以安装所需的包了。

Conda

介绍

Conda 是适用于任何语言的软件包、依赖项和环境管理系统–包括 Python, R, Ruby, Lua, Scala, Java, JavaScript, C/C++, FORTRAN 等。

Conda 也是我第一个使用的项目管理工具，它官方提供了两种版本 Anaconda 和 Miniconda 。 Anaconda 已经配置了一个完善的 Python 开发环境，而 Miniconda 就只有 Python 和 Conda 。

原理

认真来说 Conda 只是一个虚拟环境管理工具，它不为每个项目创建虚拟环境。它不直接使用官方的源，它自己维护了一系列的 channel ，保证各种包之间不会冲突。

放弃的原因

最大的原因就是它太慢了，在稍微复杂一点环境安装新的包动辄就要花费几十分钟。甚至它的 channel 之间还会起冲突，这时只能尝试调整 channel 的顺序才能安装上，再加上它的安装速度，我只能放弃使用 Conda 了。

Poetry

Poetry 是一个 Python 虚拟环境和依赖管理工具，另外它还提供了包管理功能，比如打包和发布。

Poetry 使用 PEP 518 引入的新标准 pyproject.toml 文件管理依赖列表和项目的各种 meta 信息，每个项目都有自己专属的虚拟空间，在任何电脑上只需要使用 poetry install 就可以自动配置好该项目的环境。

虽然 Poetry 更加好用，但它还是很慢，主要是计算依赖非常耗时，所以最终我也放弃使用 Poetry 。

PDM

PDM 得益于一个 2018 年的 PEP 提案（PEP582，Python local packages directory），完全摒弃了虚拟环境。也就是像 Node.js 的 npm 一样，只对包的依赖进行管理，用户不需要关心虚拟环境的问题。

它有一个简单且相对快速的依赖解析器，特别是对于大的二进制包发布。针对复杂依赖的包，它还拥有灵活且强大的插件系统，用户可以编写插件解决自己独有的问题。

总结

对于新手和 Windows 用户来说，我还是推荐 Anaconda ，因为它的用户非常多，基本任何问题都能得到社区的支持。
